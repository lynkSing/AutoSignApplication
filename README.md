# Lynk APP Auto签到

> 领克 APP 能量体任务自动化工具 — **Windows 图形界面版**

每天自动签到、查询账户、生成分享链接。双击运行，填写 `refreshToken` 和 `deviceId` 即可使用。

支持的任务：

- 每日签到
- 签到任务进度（连续 7 / 30 / 85 / 365 天）
- 能量体余额 / 累计获得
- 成长等级 / 成长值
- 分享任务（生成 H5 链接，发微信后别人点击 +5 能量体）
- 自动分享刷积分（需配置 B 账号）

## 快速开始

### 1. 获取程序

从作者处获取 **`lynkco-sign-gui.exe`**，保存到任意目录（路径尽量不要含中文或特殊字符）。

首次运行若被 Windows 拦截，点击「更多信息」→「仍要运行」。建议将程序目录加入杀毒软件信任列表。

### 2. 抓包拿凭证

用 Reqable / Charles 等工具抓包领克 APP 登录，拿到：

| 字段 | 说明 |
|------|------|
| `refreshToken` | 登录响应 `data.centerTokenDto.refreshToken`（28 天有效） |
| `deviceId` | 登录 URL 参数 `deviceId` |

详细步骤见 [抓包教程](#抓包教程必读)。

### 3. 首次运行

1. 双击 `lynkco-sign-gui.exe`
2. 阅读免责声明，勾选后点击 **「我已同意」**
3. 填写 `Refresh Token` 和 `Device ID`
4. 点击 **「保存并应用」**，进入主界面即可签到

## 图形界面说明

### 主界面

- **执行签到** — 完整签到流程
- **查询账户** — 仅查询，不签到
- **测试登录** — 验证 Token 是否有效
- **清空缓存** — 清除本地 accessToken 缓存
- **签到概览** — 连续天数、任务进度、能量体、成长值
- **运行日志** — 实时输出执行过程

顶栏 **「设置」** 可修改账号与自动化选项。

### 自动化

| 选项 | 说明 |
|------|------|
| 启动时自动查询账户 | 打开程序后自动拉取账户信息 |
| 每日自动签到 | 在指定时间窗口内随机时刻自动签到（默认 6:00–10:59） |
| 关闭时缩到系统托盘 | 关闭窗口后后台继续运行，托盘图标可恢复窗口或退出 |

### 高级选项

| 选项 | 说明 |
|------|------|
| B 账号 Token | 可选，逗号分隔，用于自动分享刷积分 |
| 分享文章 ID | 默认 `2072260486405246976` |
| 启用自动分享 | 需配置 B 账号 |
| 强制 refresh | 忽略 accessToken 缓存 |

## 抓包教程（必读）

### 工具推荐

- **iOS**：Reqable / Charles / Stream
- **Android**：Reqable / Charles / HttpCanary

### 步骤

```
1. 打开抓包工具，开启 HTTPS 录制
2. 领克 APP → 我的 → 退出登录（如已登录）
3. 重新登录（手机号 + 验证码）
4. 登录成功进入首页后停止抓包
```

> ⚠ **必须在登录前就开始抓包**，`refreshToken` 只在登录响应中返回一次。

### 找 refreshToken

搜索 `mobileCodeLogin`，查看响应 JSON：

```json
{
  "code": "success",
  "data": {
    "centerTokenDto": {
      "token": "bearer<accessToken>",
      "refreshToken": "bearer<refreshToken>"
    }
  }
}
```

拷贝 **`refreshToken`** 字段（不是 `token`）。

- `token` = accessToken，约 10 分钟有效
- `refreshToken` = 我们需要的，28 天有效

### 找 deviceId

从登录请求 URL 获取：

```
POST .../auth/login/mobileCodeLogin?deviceId=6ed00faaab514c52ab4d2f1917a040b6
```

拷贝 `deviceId` 参数值（带或不带横线均可）。

## 配置说明

程序会在用户目录自动创建配置文件：

| 文件 | 路径 |
|------|------|
| 主配置 | `%USERPROFILE%\lynk_config.json` |
| Token 缓存 | `%USERPROFILE%\.lynk_access_token.json` |

所有配置均在软件 **设置** 界面中修改，保存后写入 `lynk_config.json`，无需手动编辑。

主要配置项：

| 字段 | 说明 |
|------|------|
| `refresh_token` | 主账号 refreshToken |
| `device_id` | 设备 ID |
| `token_b` | B 账号 Token（逗号分隔，可选） |
| `content_id` | 分享文章 ID |
| `auto_share` | 是否启用自动分享 |
| `auto_sign_enabled` | 是否启用每日自动签到 |
| `auto_sign_hour_start` / `end` | 自动签到随机时间窗口（时） |
| `minimize_to_tray` | 关闭窗口时缩到托盘 |

## 常见问题

### refreshToken 多久过期？

28 天。程序运行时会自动用 refreshToken 换取新的 refreshToken。只要保持程序在后台定期运行，一般无需重抓。超过 28 天未使用则需重新抓包登录。

### accessToken 和 refreshToken 怎么区分？

都以 `bearer<UUID>` 格式出现。看来源：

- 登录响应里的 **`refreshToken`** → 填到设置
- 登录响应里的 **`token`** → accessToken，不要用

填错会报 `user_refresh_invalid_expired`，需重新抓包。

### 自动分享需要几个账号？

至少 2 个。主账号 A 生成分享链接，B 账号访问 H5 页面。B 点击后 A 获得 5 能量体 / 人 / 天。

### 关闭窗口后还在运行吗？

若开启了「关闭时缩到系统托盘」，窗口关闭后程序仍在后台，可继续自动签到。单击托盘图标可恢复窗口，右键可退出。

### 支持 Mac 或 Linux 吗？

**不支持。** 当前仅发布 Windows 版图形界面程序。
